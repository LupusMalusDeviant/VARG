01 Was ist Varg?

Keine existierende Sprache behandelt KI-Agenten als First-Class Citizens. Varg aendert das.

Python

Langsam, dynamisch typisiert, braucht venv + pip + Laufzeit. ~200x langsamer als nativ.

TypeScript

Braucht Node/Bun/Deno Runtime. ~5x langsamer. Typsicherheit zur Laufzeit entfernt.

Rust

Schnell aber verbose. Feindlich gegenueber Rapid Prototyping. Steile Lernkurve.

C#

Managed, GC-abhaengig, liefert 150MB+ Laufzeit mit. ~5x langsamer als nativ.

Warum Transpilation nach Rust?

  • Zero-Cost Abstraktionen — Rusts Optimizer uebernimmt die schwere Arbeit
  • Speichersicherheit — Borrow Checker fängt Bugs ab, die Vargs Codegen uebersehen koennte
  • Oekosystem-Zugang — use extern mappt direkt auf Cargo Crates
  • Cross-Platform — ueberall wo Rust kompiliert, kompiliert Varg
  • Bewaehrte Optimierungen — LLVM-Backend via rustc, 10+ Jahre kampferprobt

03 Kern-Features der Sprache

Agenten (First-Class Actors)

Jedes Varg-Programm ist ein Agent — ein zustandsbehafteter Actor mit Lifecycle-Hooks. Kein Library-Pattern. In die Grammatik eingebaut.

varg
agent ApiProcessor : DataProcessor {
    string last_result;
    int request_count;

    public void Init() {
        last_result = "";
        request_count = 0;
    }

    public string Process(string input) {
        request_count += 1;
        last_result = $"[{request_count}] {input.trim()}";
        return last_result;
    }

    public void Run() {
        for i in 0..5000 {
            Process($"  Request {i}  ");
        }
        print $"Processed {request_count} requests";
    }
}

Kompiliert zu einem Rust struct + impl-Block + main(). Kein Runtime-Overhead. Kein Actor-Framework. Purer kompilierter Code.

Contracts (Interface-First Design)

varg
contract DataProcessor {
    string Process(string input);
    int GetCount();
}

agent MyAgent : DataProcessor {
    // Must implement Process() and GetCount()
    // or compilation fails
}

OCAP-Sicherheit (Object-Capability Model)

Privilegierte Operationen erfordern explizite Capability-Tokens. Erzwungen zur Compile-Zeit, nicht zur Laufzeit.

varg
// Compile error — capabilities can't be constructed outside unsafe:
let token = FileSystem;  // ERROR: CapabilityConstructionOutsideUnsafe

// Correct: capabilities must be explicitly granted
public void ReadFile(string path, cap FileSystem fs) {
    var content = fs.read(path);
}

04 Token-Effizienz: Warum das fuer KI wichtig ist

Weniger Tokens = guenstigere LLM-Aufrufe. Varg ist die kompakteste typsichere Sprache fuer Agenten.

Wenn ein LLM-Agent Code generiert, liest oder analysiert, zahlt man pro Token. Jedes ueberfluessige Keyword, jeder Import, jedes self. kostet Geld. Varg wurde von Anfang an darauf optimiert, maximale Semantik pro Token zu liefern.

Warum Varg weniger Tokens braucht

Keine Imports

Python braucht from typing import Protocol, import asyncio, etc. Varg: null Zeilen Imports. Agenten, Contracts und KI-Typen sind eingebaut.

Kein self./this.

Python: self.request_count += 1
Varg: request_count += 1
5 Zeichen gespart pro Feldzugriff. Bei 20 Zugriffen: 100 Tokens weniger.

Implizite Lifecycle

Kein async def main():, kein asyncio.run(main()). Vargs Init() und Run() sind Konvention, nicht Boilerplate.

Eingebaute Syntax

retry(3) { ... } fallback { ... } statt 15 Zeilen try/except/retry-Logik mit Counter und Backoff.

Pipe-Operator: Funktionsverkettung ohne Verschachtelung

Python
# Verschachtelt, schwer lesbar:
result = format_output(
    validate(
        transform(
            parse(input)
        )
    )
)
Varg
// Linear, klar lesbar:
var result = input
    |> parse()
    |> transform()
    |> validate()
    |> format_output();

Der Pipe-Operator |> macht Datenfluesse linear statt verschachtelt. Das spart nicht nur Tokens, sondern macht den Code auch fuer LLMs einfacher zu parsen und zu generieren.

06 Agent-Entwicklung: Varg vs Python

Gleiche Logik. Verschiedene Welten.

Python
import asyncio
from dataclasses import dataclass
from typing import Protocol

class DataProcessor(Protocol):
    def process(self, input: str) -> str: ...
    def get_count(self) -> int: ...

@dataclass
class ApiProcessor:
    last_result: str = ""
    request_count: int = 0

    def process(self, input):
        self.request_count += 1
        self.last_result = \
          f"[{self.request_count}]"
        return self.last_result

    def get_count(self):
        return self.request_count

async def main():
    agent = ApiProcessor()
    for i in range(5000):
        agent.process(f"Request {i}")
    print(f"Done: {agent.request_count}")

asyncio.run(main())

    string Process(string input);
    int GetCount();

    string last_result;

    public string Process(string input) {
        request_count += 1;
        last_result =
          $"[{request_count}] {input}";
        return last_result;
    }

    public int GetCount() {
        return request_count;
    }

    public void Run() {
        for i in 0..5000 {
            Process($"Request {i}");

  • Typinferenz — var x = 42; inferiert int
  • Rueckgabetyp-Validierung — Rueckgabetypen muessen mit der Deklaration uebereinstimmen
  • Rueckgabepfad-Analyse — alle Code-Pfade muessen einen Wert zurueckgeben
  • Generic-Argumentanzahl-Validierung — Pair<int> bei 2-Parameter-Generic = Fehler
  • Contract-Compliance — Agenten muessen alle Contract-Methoden implementieren
  • OCAP-Flussanalyse — Capabilities nur in unsafe konstruierbar
  • Collection-Methodentyp-Inferenz — arr.first() gibt Element-Typ zurueck

11 Vision: VargOS

Der Compiler ist Phase 1. Das Betriebssystem ist Phase 2.

SurrealDB-Dateisystem

Vektor-indiziert, Graph-abfragbar. Jede Datei ist ein Knoten. Jede Beziehung eine Kante. Native semantische Suche.

Agenten-basierte Ausfuehrung

Agenten ersetzen Prozesse. Keine PIDs. Kein fork(). Actors mit Message-Passing und Capability-gestuetzten Ressourcen.

VRAM-Multiplexing

Mehrere LLM-Agenten teilen sich GPU-Speicher. Hibernation und Zero-Copy-Transfers zwischen Agenten.

Capability-Berechtigungen

Kein Unix rwx. Kein root. Capability-Tokens ersetzen das Berechtigungsmodell. Unfaelschbar, widerrufbar, auditierbar.

Gebaut mit Rust. Gemessen an der Industrie. Entworfen fuer das, was als naechstes kommt.
